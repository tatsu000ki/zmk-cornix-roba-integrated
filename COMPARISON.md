# オプション比較：どちらを選ぶべきか？

## オプション1: zmk-keyboard-cornixを使う

### メリット
✅ **すぐにビルドが成功する**（5分）
✅ Cornixボードの設定が既に正しい
✅ GitHub Actionsが既に動作している

### デメリット
❌ **親機（セントラル）がCornixになる**
  - Cornixが左手（親機）
  - roBaが右手（ペリフェラル）
  - **Cornixが電源を入れると、roBaが自動接続**
    
❌ リポジトリがzmk-keyboard-cornix
  - 名前が「Cornix + roBa」ではなく「Cornix」

### 実際の使用感
- Cornix（左手）の電源を入れる
- roBa（右手）の電源を入れる
- 自動的にペアリング
- **トラックボールは完全に動作**

---

## オプション2: このプロジェクト（zmk-cornix-roba-integrated）を修正

### メリット
✅ **親機（セントラル）をroBaにできる**
  - roBaが右手（親機）
  - Cornixが左手（ペリフェラル）
  - **roBaが電源を入れると、Cornixが自動接続**

✅ リポジトリ名が「zmk-cornix-roba-integrated」
  - 統合プロジェクトとして明確

### デメリット
❌ **Cornixボードの構造を完全に再設計する必要がある**
  - `boards/jzf/`構造を`boards/arm/`に変更
  - すべてのファイルを修正
  - 数時間〜数日かかる可能性

❌ ビルドが成功する保証がない
  - 複雑な修正が必要

---

## 親機（セントラル）とは？

### 親機の役割
- PCとBluetooth接続する側
- もう片方のキーボードと通信する
- **バッテリー消費が少し多い**

### ペリフェラルの役割
- 親機と通信するだけ
- PCとは直接接続しない
- **バッテリー消費が少し少ない**

---

## 実際の違い

### オプション1（Cornix親機）
```
PC ←Bluetooth→ Cornix（左手・親機）
                  ↓
                  ↓ 無線通信
                  ↓
                roBa（右手・ペリフェラル・トラックボール）
```

**使用手順:**
1. Cornix（左手）の電源ON
2. roBa（右手）の電源ON
3. 自動接続
4. トラックボール動作

### オプション2（roBa親機）
```
PC ←Bluetooth→ roBa（右手・親機・トラックボール）
                  ↓
                  ↓ 無線通信
                  ↓
              Cornix（左手・ペリフェラル）
```

**使用手順:**
1. roBa（右手）の電源ON
2. Cornix（左手）の電源ON
3. 自動接続
4. トラックボール動作

## 機能の違い

### 共通（どちらも同じ）
✅ トラックボール機能
✅ マウスレイヤー
✅ スクロールレイヤー
✅ すべてのキー機能
✅ Bluetooth接続

### 違い
- **親機がどちらか**だけ

## 推奨

### あなたの使い方次第

**roBa（右手）を先に電源ONしたい場合:**
→ オプション2（このプロジェクトを修正）

**Cornix（左手）を先に電源ONしてもOKな場合:**
→ オプション1（zmk-keyboard-cornixを使う）

**すぐに使いたい場合:**
→ オプション1（5分で完成）

**時間をかけてでも理想的な構成にしたい場合:**
→ オプション2（数時間〜数日）
